# Задача 2: Метод Хука-Дживса

## Описание

Метод Хука-Дживса — это метод прямого поиска, который объединяет два этапа: исследующий поиск вокруг базисной точки и поиск по "образцу" в направлении убывания функции.

## Алгоритм

### Этап 1: Исследующий поиск
1. Задается базисная точка xᵏ и начальное приращение Δx
2. Делается пробный шаг по каждой переменной:
   - Определяется точка x' = (x₁ᵏ + Δx₁, x₂ᵏ, ..., xₙᵏ)
   - Если f(x') < f(xᵏ), принимаем x' как новую точку
   - Иначе пробуем шаг в обратном направлении: x'' = (x₁ᵏ - Δx₁, x₂ᵏ, ..., xₙᵏ)
3. Процесс продолжается для всех координат
4. Если ни одного удачного шага не сделано, уменьшаем Δx

### Этап 2: Поиск по образцу
1. После исследующего поиска получаем точку x⁰¹
2. Направление спуска: S = x⁰¹ - x⁰
3. Минимизация функции вдоль направления S:
   x¹ = x⁰ + λS, где λ = arg min f(x⁰ + λS)

## Использование

```bash
cd task2_hooke_jeeves

```

## Параметры

- `delta0`: начальный шаг (по умолчанию 0.5)
- `epsilon`: точность (по умолчанию 1e-6)
- `alpha`: коэффициент расширения шага (по умолчанию 2.0)
- `beta`: коэффициент сжатия шага (по умолчанию 0.5)

## Тестовые функции

1. **Квадратичная функция:**
   - f₁(x) = 10(x₁ + x₂ - 10)² + (x₁ - x₂ + 4)²
   - Начальная точка: x⁰ = [0, 0]ᵀ

2. **Функция Розенброка:**
   - f₂(x) = 100(x₂ - x₁²)² + (1 - x₁)²
   - Начальная точка: x⁰ = [-1.2, 1]ᵀ

## Достоинства и недостатки

### Достоинства:
- Эффективнее метода Гаусса благодаря поиску по образцу
- Адаптивный выбор шага
- Хорошая сходимость на гладких функциях

### Недостатки:
- Может требовать настройки параметров (delta, alpha, beta)
- На некоторых функциях может быть медленным
- Не использует информацию о кривизне функции
